DMN provides open-source infrastructure for peer meditation groups.

**Not a meditation app. Not a spiritual organization. Not a business.**

**Infrastructure** - protocols, tools, systems that help people practice together sustainably.

## The Gap We're Filling

### Solo Practice Is Hard to Sustain
- 90% of people who start meditating quit within 6 months
- Willpower alone insufficient
- No accountability
- Feels isolating

### Drop-In Classes Don't Create Depth
- No commitment required = shallow engagement
- Different people every week = no group coherence
- Transactional relationship
- Can't build on previous sessions

### Teacher-Led Practice Has Limitations
- **Teachers provide value:** Technique instruction, correction, wisdom transmission
- **But:** Can create dependency if not balanced with peer practice
- **Groups need:** Both teaching (for technique) AND peer practice (for sustainability)
- **DMN fills the peer infrastructure gap** that complements teacher-led learning

**Teachers are welcome to:**
- Train node facilitators (outside sessions)
- Offer periodic intensives (separate from weekly practice)
- Provide troubleshooting support (remote)
- Participate as peers (during sessions - not as teachers)

**During DMN sessions:**
- Protocol is the teacher, not any person
- All participants are peers (including experienced practitioners)
- Teachers participate through practice, not instruction

### Existing Infrastructure Assumes Hierarchy
- All meditation infrastructure assumes teacher-student model
- No protocols for peer-facilitated groups
- No tools for distributed networks
- Organizations centralize control

**We're building the missing piece: peer-to-peer meditation infrastructure.**

## Our Solution

### 1. Peer Structure
**Everyone equal:**
- No teachers or gurus during sessions
- Facilitation rotates among members
- No authority positions
- Decisions made collectively

**Experienced practitioners:**
- Participate as peers (not teachers)
- Act as field anchors/amplifiers through practice
- Support through presence, not instruction

**Why this works:**
- Prevents dependency
- Distributes responsibility
- Builds autonomy
- Resilient (doesn't collapse if one person leaves)

### 2. Weekly Commitment
**Regular practice, long-term:**
- Same time, same place, every week
- Commitment to the group (not just yourself)
- Build on previous sessions
- Depth emerges over months/years

### 3. Heart-Centered Practice
**Compassion meditation (research-validated):**
- Self-compassion
- Loved ones
- Gratitude
- Silent integration

**Based on research:**
- Heart-brain coherence effects
- Heart rate variability synchronization
- Positive emotions enhance coherence
- Group practice amplifies individual effects

**Technique:**
- Heart-centered breathing (generic, research-backed)
- Mixed affirmation/intention phrases
- Framework-neutral (accessible to all)

### 4. Open Source
**Everything public and forkable:**
- Protocols documented on GitHub
- Tools are open-source code
- Anyone can adapt/modify
- No proprietary methods

### 5. Distributed Network
**Autonomous nodes, not centralized organization:**
- Each node operates independently
- Nodes connect horizontally (peer-to-peer)
- No central authority
- Network resilient to single points of failure
